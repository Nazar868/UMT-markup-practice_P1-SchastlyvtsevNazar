# Flora — Floral Artistry

Семантична, адаптивна HTML/CSS-верстка сторінки квіткової майстерні **Flora** з інтерактивним шаром: скоуп 1 (заняття 1–12) + скоуп 2 (заняття 13–20).

## Демо

Після деплою додайте посилання на GitHub Pages сюди та в секцію **About** репозиторію. Оскільки дані підвантажуються з локального `json-server`, для повноцінної перевірки каталогу, форм і пагінації потрібно запустити mock API локально (інструкція нижче) — на GitHub Pages без запущеного API каталог покаже стан помилки завантаження.

## Структура проєкту

```
├── index.html
├── db.json                     # дані для json-server (bouquets / requests / subscribers)
├── package.json                # скрипт "api" для запуску json-server
├── css/
│   ├── modern-normalize.css    # скидання стилів
│   └── styles.css              # усі стилі проєкту, CSS-змінні, адаптив, модалка, форми
├── images/
│   ├── icons.svg                # SVG-спрайт іконок (<use>)
│   └── *.jpg                    # оптимізовані фото (варіанти @1x / @2x)
├── js/
│   ├── main.js                  # мобільне меню (is-open), AOS-анімації
│   ├── modal.js                 # модалка заявки + форма підписки (axios, async/await)
│   └── catalog.js               # динамічний каталог: axios, пагінація, фільтрація, пошук
├── favicon.ico
└── favicon-32.png
```

## Запуск локального mock API (json-server)

Каталог букетів, форма заявки та форма підписки працюють через локальний `json-server`, що читає `db.json`.

```bash
npm install       # встановить json-server (devDependency)
npm run api       # підніме API на http://localhost:3001
```

Після цього відкрийте `index.html` (наприклад, через розширення Live Server) — базова адреса API налаштована в `<head>` сторінки через `window.FLORA_API = "http://localhost:3001"`.

Доступні ендпоінти:

- `GET /bouquets?_page=1&_limit=6&category=roses&q=rose` — список букетів із пагінацією, фільтром за категорією та пошуком.
- `POST /requests` — заявка з модального вікна.
- `POST /subscribers` — підписка з форми у футері.

## Що реалізовано (скоуп 1, заняття 1–12)

- Семантична розмітка: `header`, `main`, `footer`, один `nav`, один `h1`, `h2` для секцій, списки в `ul`.
- Шрифти Fraunces + Work Sans з Google Fonts (одне посилання), `font-family` заданий лише на `body`.
- Кольори винесені в CSS-змінні (`:root`), підключений modern-normalize, `box-sizing: border-box`.
- Контейнер `.container` (1336px), відступи через `margin`/`padding`, flexbox — там, де потрібен, без фіксованих висот.
- Mobile-first адаптив на 375 / 768 / 1440 px, без горизонтальної прокрутки від 320 px.
- SVG-іконки зібрані у спрайт `images/icons.svg`, підключені через `<use>`.
- Переходи 250ms `cubic-bezier(0.4, 0, 0.2, 1)` на інтерактивних елементах, явні властивості (без `all`).
- Підключена бібліотека анімацій AOS (`data-aos` атрибути).
- Мобільне меню на всю висоту в’юпорту, приховане за замовчуванням, відкривається класом `is-open`.
- Без `!important`, без console-помилок/логів.

## Що додано (скоуп 2, заняття 13–20)

- **Ретинізація**: контентні `<img>` мають `srcset` з `@1x`/`@2x`; фонове зображення hero підвантажується через `@media (min-resolution: 2dppx)`.
- **Модальне вікно** (`#orderModal`): бекдроп на весь вʼюпорт, фіксоване позиціонування, центрування, закриття по кнопці/бекдропу/`Esc`, стан `is-open`, блокування скролу `body` (`no-scroll`).
- **Форми**: заявка в модалці та форма підписки у футері — семантичні `label`/`input`/`name`/`type="submit"`/`placeholder`; чекбокс ліцензійної угоди реалізований вручну через SVG-спрайт (`icon-check`), нативний `input` прихований через `.checkbox__input`.
- **API-запити**: `axios` (CDN) + `async/await`, дані з `json-server`; каталог букетів формується повністю динамічно через шаблонні рядки та єдиний виклик `insertAdjacentHTML` (без `appendChild` у циклі, без статичної розмітки списку); обробка `loading`/`error`-станів.
- **Пагінація та фільтрація**: кнопка «Load More» додає `page`/`limit` до запиту; пошук (`q`) і фільтр за категорією скидають `page` на `1`; стан застосунку (`page`, `limit`, `category`, `query`, вже відрендерені id) зберігається в одному обʼєкті `state` у `catalog.js`; дублювання карток виключено (`state.renderedIds`); порожня відповідь і завершення колекції показують відповідне повідомлення, кнопка ховається.

## Деплой на GitHub Pages

1. Перейменуйте репозиторій у формат `UMT-markup-practice_P2-ПІБ` (або доповніть існуючий репозиторій цим кодом).
2. Запуште вміст цієї папки в гілку `main`.
3. У `Settings → Pages` виберіть джерело `main` / `root` і збережіть.
4. Додайте отримане посилання GitHub Pages у секцію **About** репозиторію.
5. Оскільки на GitHub Pages немає бекенду, до посилання варто додати `db.json` окремим файлом і зазначити в описі, що для перевірки динаміки потрібно підняти `json-server` локально (`npm install && npm run api`).
6. Видаліть усі робочі гілки, окрім `main` та `gh-pages`.
